Introduction

Type coercion is the process of converting a value from one type to another.

Data types

To know what type coercion is, we must first understand values. Values are properties we assign to variables (i.e. let num = 3).

We know that there are several built-in types of values in Javascript. Six of them are primitive value types:

  • number
  • boolean
  • string
  • null
  • undefined
  • symbol (new in ECMAScript 6)

Meanwhile, an object (and hence functions and arrays) is a more complex data structure.

Sometimes, in order for values in a program to be operated on, the corresponding data types need to be converted. This is where type coercion comes into play.

Explicit and Implicit Type Coercion

When the intention to convert a value to another type is expressed in the code, it is referred to as explicit type coercion. It is also known as type casting.

On the other hand, implicit type coercion, or simply type coercion, is the behavior that JavaScript exhibits where a value is automatically converted from one type to another, even when it is not explicitly expressed in the code.

Possible Uses

For the vast majority of the time, you will not want to use type coercion. Type coercion can be confusing and unpredictable, leading to uninteded consequences in your program and can be difficult to debug as well.

Nonetheless, type coercion has some advantages as well. It can help simplify code and make it more readable. A good JavaScript developer would take advantage of type coercion's usefulness and also avoid it when unnecessary and risky.

You need the knowledge of what is happening under the hood to understand why you are or are not getting the outcome you seek with your code. Learning more about JavaScript's data types, values, and type conversion facilitate the writing of good and clean code.

Examples

    // "51"
  • Coercing to Boolean in Conditional Statements:
    var x = "string";

    if (x) {
    console.log("Boolean (true)")
    } else {
    console.log("Not Boolean")
    }
    // Boolean (true)
  • Abstract Equality Comparison:
    var x = "100";
    var z = 100;

    x == z;
    // true
